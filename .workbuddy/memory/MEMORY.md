### 校对铁律（2026-06-22 用户明确要求）
1. **所有时间必须以北京时间为准（GMT+8）**：信息源如用UTC/EST等时区，必须换算为北京时间后写入
2. **数据准确性是校对核心**：赛程表、对战双方、比分——这三项如有错误必须立即修正，与FIFA官方交叉核对
3. 2026世界杯有12个小组(A-L)，每组4队，共48队。小组归属错误是常见问题
4. **文件命名日期以北京时间为准（不可协商）**：文件名中的 YYYY-MM-DD 必须与北京时间日期一致（对照 schedule.json）。例如：比赛在北京时间6月23日凌晨开赛，文件名应为2026-06-23而非美东时间的6月22日。不一致时必须重命名修正。**这条规则在生成/校对每个文件时必须手动核查**
5. **schedule.json不是绝对权威，Excel赛程表是最终仲裁源**：如 schedule.json 与 `2026FIFA世界杯赛程表_完整版.xlsx`（位于 `D:/我的坚果云/OB笔记/自媒体/fwc2026/content/new/`）不一致，以 Excel 为准。Excel 含 ET→北京时间转换、场馆、城市等完整信息。2026-06-23 发现旧 schedule.json 有系统性日期偏移（第2场韩国vs捷克日期错1天 → 连锁到所有后续比赛），已完全从 Excel 重建。
6. **预测比分/备选比分 一致性规则（2026-06-23 用户明确要求）**：报告的"最可能比分"应在 match_data.json 中为主预测（prediction），备选1为备选比分（alt_score）。两者不能写反。生成报告后必须核对概率：概率最高的比分→最可能/主预测，次高的→备选1/备选比分。巴拿马vs克罗地亚曾出现主备互换错误（概率20%的3-0写了备选，概率15%的2-0写了最可能）
7. **概率表方向必须与 schedule.json 一致（2026-06-24 校对发现）**：build_site.py 解析 `<th>{X}胜</th><th>平局</th><th>{Y}胜</th>` 时，**第 1 列固定为 team_a 胜率、第 3 列为 team_b 胜率**，与报告作者写的胜方是谁无关。报告生成时如果按"胜方在左"思维把"巴西胜 71%"放在第 1 列，但 schedule.json 中 team_a 是苏格兰，则会导致 index.html 首页胜率数字颠倒。**生成报告时必须确认概率表第 1 列 = schedule.json 的 team_a**。2026-06-24 校对发现 3 份报告（scotland-brazil/czech-mexico/south-africa-south-korea）此错误，已全部修复。
8. **墨西哥时区特殊处理（2026-06-24 校对发现）**：墨西哥已取消夏令时，Mexico City/Monterrey/Guadalajara 全年使用 CST (UTC-6)，与美国 CT (UTC-5) **不同**。ET 21:00 对应墨西哥 CST 是 19:00（差 2h），不是 21:00。生成报告时如果用 ET 时间作"开球时间"再标注「CST」是错误的，必须做时区转换。Excel 权威源中墨西哥场次都标了 ET 时间，报告需自行换算。2026-06-24 校对发现 3 份报告（morocco-haiti 写成 15:00 ET 实为 18:00 ET；czech-mexico/south-africa-south-korea 写成 21:00 CST 实为 19:00 CST）此错误，已全部修复。

### 铁律7：比分方向必须与卡片球队位置一致（2026-06-23 用户明确指正）
- **网站卡片**：左侧=team_a（schedule.json），右侧=team_b
- **预测比分格式**："TeamX N-M TeamY"（N是TeamX的得分，M是TeamY的得分）
- **显示规则**：必须将比分映射为 `team_a得分 : team_b得分`，不能机械提取
- **正例**：预测"克罗地亚 3-0 巴拿马"，卡片"巴拿马 vs 克罗地亚" → 显示 `0 : 3`
- **反例**：预测"克罗地亚 3-0 巴拿马"，卡片"巴拿马 vs 克罗地亚" → 显示 `3 : 0` ← 严重错误！
- **备选比分同理**：方向跟随主预测
- index.html 已内置 `formatPredScore()` / `formatAltScore()` 自动处理方向，但生成预测报告时应尽量让 TeamX=team_a 以减少歧义

### 铁律9：MD3 报告"末轮"概念必须严格（2026-06-24 用户指正，6/25 报告全面返工）
- **6/25 6 场报告 = 小组赛 MD3（第 3 轮）即末轮**，**没有第 4 场**，"末轮"指向未来是严重逻辑错误
- 算分公式 "本场+X分(末轮)=Y 分" **全部错**——本场结束=小组赛结束，本场×1=最终积分
- **常见错例**（6/25 报告中出现的）：
  - "摩洛哥末轮对苏格兰" → 错。摩洛哥 MD3 = 对海地（本场），不会再踢苏格兰
  - "本场若赢(6+1=7分)" → 错。6+3=9 分
  - "末轮对韩累黄停赛" → 错。累黄停赛只影响本场或带进淘汰赛
  - "为末轮对X留力" → 错。本场就是末轮，留力意义减弱
- **正确表述**：
  - "本场是小组赛收官战（MD3）" / "本场后小组赛结束"
  - "赢→9分小组第一（锁定）" 而非"赢+X分(末轮)=Y分"
  - "累黄再吃黄停赛" 而非"末轮停赛"
- **核心检查清单**（每份 MD3 报告必查）：
  1. 开篇形势段"末轮"用法 → 应指 MD3 即本场
  2. "末轮对X" → 删除（无第 4 场）
  3. 算分公式"本场+末轮=X+Y分" → 改为"本场=X分（MD3 收官）"
  4. "为末轮准备" → "为淘汰赛留力"或"为下届赛事锻炼"
  5. 表格"末轮"列 → 改为"出线形势"或"本场后积分"
- 6/22 等 MD2 报告里的"末轮对X"指 MD3 = **正确**（因 MD2 后还有 MD3 末轮）
