## 任务描述

基于最新可获得的数据，预测 2026 年世界杯「明天所有比赛（含今天已开始但未结束的）」的赛果和可能比分。

## 执行清单（新 AI 按此顺序操作）

### 0. 加载铁律

在开始任何工作前，读取并记住 [PIPELINE.md 的七条铁律](PIPELINE.md#七条铁律最高优先级)。

### 1. 赛程确认

1. 读取 `schedule.json`，找出「明天」的比赛（北京时间日期）
2. 用 FIFA 官方赛程 (fifawc-2026.com/full-match-schedule/) 交叉核对
3. 如不一致，以 FIFA 官方为准，标注差异
4. 确定文件名日期（北京时间）

### 2. 数据采集（12 个维度）

对每场比赛执行以下采集：

| # | 维度 | 说明 |
|---|------|------|
| 2.1 | 球员覆盖 | 每队 16-20 人：首发 11 人 + 5-9 名轮换/替补 |
| 2.2 | 对手强度校准 | 按 FIFA 排名分 4 档 (Tier 1-4) |
| 2.3 | xG 量化 | 近 10 场 xG/xGA，预测本场 xG 区间（禁止只写"大球/小球"）|
| 2.4 | 门将分析 | 扑救率、PSxG+/- |
| 2.5 | 首发不确定性 | 老将/伤愈标注首发概率 (高/中/低) |
| 2.6 | 犯规率/纪律 | 场均犯规 ≥15 = "高犯规率→节奏破坏器" |
| 2.7 | 联赛→大赛折扣 | 首秀 ×0.70-0.80 / 有经验非核心 ×0.85-0.95 / 老将 ×0.90-1.00 |
| 2.8 | 教练评估 | 8 维度 1-10 评分，分级 S/A/B/C |
| 2.9 | 替补攻击力 | 替补进球率 + "替补奇兵"候选人 |
| 2.10 | 逆转因子 | 历史逆转频率 |
| 2.11 | 场地/环境 | 海拔 >1000m 重点分析 |
| 2.12 | 教练视角 | 同组四队全覆盖，第一人称分析 |

**数据源优先级**：
1. LiveScores.com — 近期赛果、阵容、伤停
2. FIFA 官网 — 赛程、排名、球队名单
3. ESPN/WhoScored — 球员表现、xG 数据、评分

### 3. 分析（遵守概率规则）

**禁止项**：
- 禁止线性推演、概率平均主义 (33/34/33)
- 禁止模糊"大球/小球"
- 禁止仅凭名气排名预测
- 必须考虑：早期进球后形态变化、防守崩溃、犯规节奏破坏

**概率规则**：
- 胜率差异 ≥ 8%（如 42/30/28）
- 平局 ≥ 30% 且胜方 < 45% → 平局为最可能单一结果
- 信号强度 ≥ 3 条强信号 → 胜率 ≥ 55%

**不确定性**：
- 区分"数据支持较强"和"主观推测"
- 缺失数据 → 降低置信度
- 1 个最可能比分 + 2-3 个备选比分，均标注概率

## 关于备选比分格式的重要说明

**为什么必须同时有 table.score 和段落格式？**

| 格式 | 用途 |
|------|------|
| `<table class="score">` | 报告中可读的表格展示 |
| `<p>备选比分1：X-X</p>` | `build_site.py` 解析 → `match_data.json` → 首页卡片显示 |

如果只有表格而没有段落格式，首页卡片不会显示备选比分。

## 关于预测比分方向的说明

**铁律6 的由来（2026-06-23 事故）**：

- 预测"克罗地亚 3-0 巴拿马"（克罗地亚赢）
- `schedule.json` 中 team_a=巴拿马，team_b=克罗地亚
- 如果只写"3-0"纯数字 → 首页显示"巴拿马 3:0 克罗地亚" ← 完全反了！
- 正确做法：写"克罗地亚 3-0 巴拿马" → `formatPredScore()` 自动识别并映射为 `0 : 3`

**所以，table.score 中每个比分都必须写完整格式"X队 N-M Y队"，备选比分段落格式中也必须如此。**
