### 第 2 步：数据准确性校验（最高优先级）

这是校对的核心环节。必须通过 WebSearch/WebFetch 交叉核对 FIFA 官方或权威数据源。

#### a. 赛程表核对

| 检查项 | 方法 |
|--------|------|
| 比赛日期是否与 FIFA 官方一致 | 对比 fifawc-2026.com |
| 比赛时间是否标注为北京时间 | 检查时区转换是否正确 |
| 比赛场地/城市是否正确 | 对比官方赛程 |
| 时区标注是否精确 | 北美 4 时区不可统一，墨西哥全年 CST |

#### b. 对战双方核对

| 检查项 | 方法 |
|--------|------|
| 对阵双方是否与 FIFA 官方一致 | 对比官方赛程 |
| 小组分组是否正确 (A-L 组，每组 4 队) | 核对 FIFA 官方分组 |
| 同组球队只与本组其他 3 队比赛 | 不会跨组对阵 |

#### c. 比分核对（仅已完成的比赛）

- 报告中引用的历史比分是否与 FIFA 官方结果一致
- 与至少一个权威源（FIFA官网、ESPN、BBC Sport）交叉验证

#### d. 积分表核对

- 小组积分表是否基于正确的首轮赛果
- 球队排名、进球数、失球数、积分是否一致

#### e. 球员数据核对

- 年龄、位置、俱乐部归属是否正确
- 与 FIFA 官方名单或权威数据源一致

**处理原则**：发现错误 → 立即用 Edit 工具修正

### 第 3 步：内容质量提升与重写

**这是从普通校对提升到出品级质量的关键步骤。**

#### 3.1 可读性与表达
- 删除重复论述，合并相似观点
- 交替使用短句和长句，关键结论用短句强调
- 每个数据指标后面必须有"这意味着什么"
- 删除"值得关注""不容小觑""拭目以待"等套话

#### 3.2 论据强度
- 每个预测结论是否有 ≥2 条独立支撑论据
- 缺少支撑 → 补充或标注置信度降低
- 论据链条断裂 → 补全推理链

#### 3.3 数据呈现
- 关键数据是否有可视化描述
- 对比性数据是否并列呈现
- 数据是否有上下文锚点

#### 3.4 教练视角 (2.12 节) 特别检查
- 是否覆盖了同组全部 4 队
- "我是 X 队主教练——"第一人称口吻是否自然
- 是否包含具体的针对性策略和 B 计划

#### 3.5 最终结论质量
- 明确的"最可能比分 + 备选比分"且标注了概率
- 结论是否与前面的分析一致
- 是否存在前后矛盾或逻辑跳点

**改写原则**：严禁改动预测结果本身（胜率、比分、xG 区间保持原样），只提升表达方式和呈现质量。

### 第 5 步：格式规范检查

**HTML 报告**：
- 完整的 `<!DOCTYPE html>` 和闭合标签
- 蓝色 `#0033A0` 主色调
- 页脚存在且不含违禁内容
- 时间格式：`数据更新时间：YYYY-MM-DD HH:MM`（北京时间）
- 比赛时间标注：`⏰ 当地时间 时区 (城市当地时间) / 北京时间`

**MD 报告**：
- 纯 Markdown 格式（无 HTML 标签）
- 文件名符合 `YYYY-MM-DD-teamA-teamB-prediction.md` 规范
- 章节结构完整（比赛信息→...→最终结论）
- 文件内日期标注：`YYYY-MM-DD HH:MM 北京时间 (当地时间HH:MM)`

---

### 第 6 步：数据一致性快速校验

- HTML 和 MD 版本中预测结果是否一致（胜率、比分）
- 胜率三档总和是否在 97%-103% 范围
- 三档接近（差异均 <8%）→ 标注"概率差异不足，置信度可能偏高"

---

### 第 7 步：备选比分 & 推演比分 & 比分方向检查

#### 7.1 备选比分格式检查

检查每个 HTML 报告是否同时包含：
1. 段落格式：`<p>备选比分1：X-X（概率 ~XX%）</p>`
2. 表格格式：`<table class="score">...<td>备选1</td>...`

**只有表格格式 → 必须补充段落格式**

#### 7.2 推演比分覆盖检查

运行 `python build_simulation_data.py` → 检查 `simulation_scores.json` 是否包含当天所有比赛。

缺少数据时的修复流程：
1. 检查 Dixon-Coles MD 文件是否存在
2. 队名是否已翻译为中文（参考 `EN_TO_ZH_TEAM` 映射表）
3. 推荐精准比分格式是否正确（`**X-X**` 不带球队名后缀）
4. 修复后重新运行脚本

#### 7.3 预测比分方向检查（⚠️ 2026-06-23 新增，不可跳过）

运行 `python build_site.py` 生成 `match_data.json` 后：

1. 检查每场比赛的 `prediction` 字段
   - 是否包含完整球队名（"克罗地亚 3-0 巴拿马"而非仅"3-0"）
   - 纯数字 → 回到 HTML 修正 table.score 格式

2. 核对备选比分方向
   - alt_score 数字是否与主预测的胜负方一致
   - 例：主预测"克罗地亚 3-0 巴拿马"→ 克罗地亚进 3 球
   - 备选"2-0"也应表示克罗地亚进 2 球

3. 检查概率排序
   - table.score 中概率最高的 → 最可能/主预测
   - 次高 → 备选 1

## 校对原则总结

| 优先级 | 内容 | 处理方式 |
|--------|------|----------|
| 🔴 最高 | 赛程/对阵/比分准确性 | 与 FIFA 官方核对，发现错误立即修正 |
| 🔴 最高 | 时间必须为北京时间 | 非北京时间必须转换 |
| 🔴 最高 | 文件命名日期 | 对照 schedule.json 核对，不一致重命名 |
| 🔴 最高 | 预测比分带完整球队名 | 纯数字 → 修正为 "X队 N-M Y队" 格式 |
| 🟡 高 | 备选比分段落格式 | 缺少则补充 |
| 🟡 高 | 推演比分全覆盖 | 缺失则修复 |
| 🟢 中 | 违禁词 | 立即删除/替换 |
| 🟢 中 | 格式问题 | 自动修复 |
| 🔵 低 | 内容质量 | 改写但不改预测数值 |
| ⚪ 参考 | 战术分析合理性 | 只标注，不修改 |
